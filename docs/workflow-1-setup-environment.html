Seven Core Workflow
Workflow 1

Setup Environment Parameters

Thiết lập bộ tham số môi trường trước khi chạy test API

Workflow 1 là bước tạo nên "bộ điều kiện chạy" cho toàn bộ hệ thống. Ở bước này, người dùng khai báo API sẽ được gọi ở đâu, dùng tài khoản/token nào, headers nào cần gửi kèm, và những giá trị nào sẽ được tái sử dụng trong các test case về sau.

Environment
Parameters Base URL, auth, headers, variables, secrets và runtime context.

Overview nhanh

Mục tiêu

Đảm bảo mỗi lần generate hoặc execute test đều dùng đúng server, đúng token, đúng header và đúng biến đầu vào.

Đầu vào

Tên môi trường, base URL, thông tin xác thực, headers mặc định, variables, giá trị bí mật và tùy chọn timeout/retry.

Đầu ra

Một environment profile sẵn sàng được Workflow 5, Workflow 6 và Workflow 7 sử dụng khi giải thích tham số, sinh test và chạy test.

Các bước chạy chuẩn của Workflow 1

Chọn project và môi trường

Người dùng chọn project đang làm việc, sau đó tạo hoặc chọn environment như Local, Development, Staging hay Production.

Khai báo base URL

Nhập địa chỉ gốc của API. Tất cả endpoint được import từ Swagger sẽ được gắn vào địa chỉ này khi cần gọi thật.

Thiết lập xác thực

Chọn cách xác thực phù hợp: bearer token, API key, basic auth hoặc header riêng theo hệ thống đang test.

Thêm headers mặc định

Khai báo các headers cần gửi trong hầu hết request, ví dụ content type, tenant, language hoặc correlation id.

Định nghĩa variables

Tạo các giá trị dùng lại nhiều lần như userId, productId, email test, password test, tenantId hoặc mã đơn hàng mẫu.

Đánh dấu secret

Những giá trị nhạy cảm như token, password và API key được xem là secret để tránh hiển thị công khai trong màn hình và report.

Kiểm tra khả dụng

Hệ thống có thể chạy một lần check nhẹ để biết base URL có truy cập được không và thông tin xác thực có hợp lệ không.

Lưu và kích hoạt

Khi hợp lệ, environment được lưu lại và có thể đặt làm mặc định cho các lần generate/execute test tiếp theo.

Bên trong workflow hoạt động ra sao?

1. FE gửi cấu hình

Màn hình Environments gọi API execution-environments với tên môi trường, base URL, variables, headers và auth config.

2. Backend validate

Controller dispatch command để kiểm tra project, quyền người dùng, URL, rowVersion, default flag và dữ liệu auth trước khi lưu.

3. Lưu execution environment

Cấu hình hợp lệ được lưu thành ExecutionEnvironment; nếu đặt default, hệ thống bảo đảm project chỉ có một môi trường mặc định.

4. Resolve khi chạy test

Khi start test run, backend lấy environment được chọn hoặc default, rồi resolve base URL, headers, auth và variables thành runtime context.

5. Bơm vào request thật

Runtime context được đưa vào variable bag để workflow sau thay placeholder, ký auth header và ghép URL trước khi gọi API.

Luồng code và bảng dữ liệu

  • FE: EnvironmentsPage dùng environmentService gọi /api/projects/{projectId}/execution-environments để tạo, sửa, xóa hoặc lấy environment.
  • Controller: ExecutionEnvironmentsController nhận request rồi dispatch AddUpdateExecutionEnvironmentCommand hoặc DeleteExecutionEnvironmentCommand.
  • Persist: cấu hình được lưu trong testexecution.ExecutionEnvironments, gồm baseUrl, variables, headers, authConfig, isDefault và rowVersion.
  • Khi execute: StartTestRunCommandHandler đọc testexecution.ExecutionEnvironments theo EnvironmentId hoặc default environment của project.
  • Runtime: TestExecutionOrchestrator resolve environment thành variable bag; dữ liệu run được ghi thêm vào testexecution.TestRuns với EnvironmentId đã chọn.

Vai trò trong luồng tổng thể

Workflow 1 dùng trước khi phân tích và chạy test. Nếu Swagger cho biết API có endpoint nào, thì environment cho biết endpoint đó sẽ được gọi ở đâu và cần điều kiện nào để gọi thành công. Vì vậy, nó là nền tảng cho việc giải thích tham số, sinh test case và execute test case.
Environment không chỉ là một form cấu hình. Nó là bộ nhớ ngữ cảnh của một lần chạy test: mỗi request trong suite sẽ tham chiếu vào đây để biết nên dùng server nào, đăng nhập bằng gì, và lấy biến đầu vào từ đâu.

Điều kiện để xem là cấu hình đúng

  • Có ít nhất một environment đang hoạt động cho project.
  • Base URL đúng định dạng và trỏ đến đúng hệ thống cần test.
  • Thông tin xác thực phù hợp với API đang sử dụng.
  • Headers mặc định không gây xung đột với từng request riêng lẻ.
  • Variables có tên rõ ràng, dễ đọc và không bị trùng.
  • Nếu dùng sai base URL, test có thể fail hàng loạt dù API contract vẫn đúng.
  • Nếu token hết hạn, lỗi sẽ nằm ở authentication chứ không phải logic nghiệp vụ.
  • Nếu thiếu variables, Workflow 5 và Workflow 7 có thể không resolve được tham số.
  • Nếu để lộ secret trong report, dữ liệu nhạy cảm có thể bị chia sẻ ngoài ý muốn.
  • Nếu không check environment trước, lỗi thất bại sẽ khó phân biệt với lỗi API.
Workflow 1 tạo nên execution context. Các workflow sau không cần hỏi lại những thông tin nền tảng này, mà chỉ việc lấy context đã được cấu hình để tiếp tục phân tích, sinh test và chạy test.
