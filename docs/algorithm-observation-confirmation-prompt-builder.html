Thuật toán dựng prompt cho LLM

Observation-Confirmation prompting

Thuật toán này tạo prompt hai pha để LLM sinh expectation và scenario dựa trên bằng chứng từ OpenAPI/SRS. Mục tiêu là giảm hallucination: trước tiên quan sát constraint, sau đó xác nhận từng constraint bằng evidence.

observe
confirm
output json

Nằm ở đâu và được gọi khi nào?

Implementation

ObservationConfirmationPromptBuilder implement IObservationConfirmationPromptBuilder.

Điểm gọi chính

HappyPathTestCaseGenerator, LlmScenarioSuggester và TestGenerationPayloadBuilder gọi BuildForSequence.

Vai trò workflow

Prompt được đóng vào n8n payload để LLM sinh test case hoặc scenario có cấu trúc, có evidence và bám sát spec.

Đầu vào và đầu ra

Đầu vào

  • EndpointPromptContext: method, path, operationId, summary, description.
  • Parameter, response, request/response schema, examples từ OpenAPI.
  • BusinessContext từ rule nghiệp vụ hoặc SRS của suite.

Đầu ra

  • SystemPrompt định nghĩa vai trò QA engineer và format JSON.
  • ObservationPrompt yêu cầu liệt kê constraint.
  • ConfirmationPromptTemplate yêu cầu xác nhận constraint bằng evidence.
  • CombinedPrompt dùng cho single-shot model.

Luồng thuật toán cho một endpoint

Nhận context

BuildForEndpoint trả null nếu context rỗng, tránh tạo prompt không có dữ liệu.

Dựng spec block

BuildSpecBlock gom method, path, parameter, schema, response, example và business rules.

Tạo observation prompt

Pha 1 yêu cầu LLM liệt kê mọi constraint testable: type, format, required, enum, range, relation, business rule.

Tạo confirmation prompt

Pha 2 yêu cầu giữ constraint có evidence, loại constraint suy diễn hoặc mâu thuẫn với example.

Tạo combined prompt

Prompt một lượt mô phỏng observe, confirm và output JSON cho model hoặc webhook không chạy multi-turn.

Gắn system prompt

System prompt bắt output là JSON array gồm field, constraint, type, source, evidence, confidence và assertion.

Build sequence

BuildForSequence xử lý endpoint đã được order và giữ danh sách previous endpoints.

Thêm cross context

Nếu endpoint hiện tại liên quan path với endpoint trước, prompt được bổ sung ngữ cảnh dữ liệu liên endpoint.

Cách được dùng trong workflow sinh test

Happy path generation

HappyPathTestCaseGenerator.GenerateAsync lấy ordered metadata, map sang EndpointPromptContext, gọi BuildForSequence, rồi đưa prompt vào n8n payload.

Boundary/negative suggestion

LlmScenarioSuggester chỉ dùng prompt builder khi profile bật UseObservationConfirmationPrompting; nếu thiếu prompt thì fallback sang prompt tự dựng.

Khi debug cần nhìn gì?

  • Nếu LLM sinh constraint không có bằng chứng, kiểm tra SystemPrompt và ConfirmationPromptTemplate có được gửi trong payload không.
  • Nếu prompt quá dài bị cắt, kiểm tra đoạn truncate trong LlmScenarioSuggester hoặc TestGenerationPayloadBuilder.
  • Nếu thiếu context liên endpoint, kiểm tra thứ tự endpoint đầu vào và logic IsLikelyRelated dựa trên path segments.
