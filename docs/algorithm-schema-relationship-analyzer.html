Thuật toán phát hiện quan hệ schema

Schema relationship analysis

Thuật toán này đọc các $ref trong OpenAPI schema để tìm dependency ẩn giữa endpoint. Nó giúp biết endpoint nào tạo ra resource mà endpoint khác cần dùng, kể cả khi quan hệ đó đi qua nhiều lớp schema.

Nằm ở đâu và được gọi khi nào?

Implementation

SchemaRelationshipAnalyzer implement ISchemaRelationshipAnalyzer.

Điểm gọi chính

ApiTestOrderAlgorithm.FindSchemaSchemaEdges gọi analyzer để bổ sung cạnh dependency trước khi sort.

Vai trò workflow

Workflow order dùng cạnh SchemaSchema để tránh sinh hoặc chạy test theo thứ tự làm mất dữ liệu tiền đề.

Ba nhóm việc chính

Dựng graph trực tiếp

BuildSchemaReferenceGraph map schema name sang các schema mà payload của nó tham chiếu.

Tính closure

ComputeTransitiveClosure mở rộng quan hệ: nếu A cần B và B cần C, A được xem là liên quan tới cả B và C.

Tạo dependency edge

FindTransitiveSchemaDependencies và FindFuzzySchemaNameDependencies tạo cạnh giữa operation consumer và producer.

Luồng thuật toán transitive schema

Thu schema refs

Từ endpoint metadata, lấy ParameterSchemaRefs, ResponseSchemaRefs và schema payload.

Extract $ref

Regex tìm #/components/schemas/X hoặc #/definitions/X trong payload.

Dựng cạnh schema

Nếu schema A chứa ref tới B, graph có cạnh A sang B. Self-reference bị bỏ qua.

Tính bắc cầu

Warshall-style closure thêm các quan hệ gián tiếp để tìm dependency qua nhiều lớp nested schema.

Map producer

Dựng reverse map: schema response nào được operation nào produce.

So consumer với producer

Nếu parameter schema của consumer đi tới schema mà producer trả ra, tạo cạnh SchemaSchema.

Fuzzy schema name

Strip prefix/suffix như Create, Request, Response, Dto để match base name.

Trả về edges

Cạnh transitive có confidence 0.85, fuzzy name có confidence 0.65.

Ví dụ cách hiểu

Transitive dependency

OrderRequest ref tới OrderItem, và POST /items trả về OrderItem. Endpoint dùng OrderRequest có thể cần POST /items chạy trước.

Fuzzy dependency

CreateUserRequest và UserResponse cùng base name User. Analyzer xem đây là quan hệ có độ tin cậy trung bình.

Khi debug cần nhìn gì?

  • Nếu thiếu dependency, kiểm tra payload có thật sự chứa $ref theo pattern được regex hỗ trợ không.
  • Nếu dependency quá rộng, kiểm tra fuzzy base name có strip prefix/suffix quá mạnh không.
  • Trong ApiTestOrderAlgorithm, hiện code gọi BuildSchemaReferenceGraphLegacy từ payload collection, nên độ chính xác phụ thuộc vào payload được metadata service cung cấp.
